# 微信小程序开发设计文档

## 一、项目概述

### 1.1 项目背景
将现有课程网站（Flask/Python）和电商网站（PHP）重构为统一的微信小程序，同时保留PC端入口供管理员使用。小程序端不提供管理员入口，仅面向普通用户。

### 1.2 现有系统分析

#### 课程网站（course-platform）
- **技术栈**: Python Flask + SQLAlchemy + MySQL + Redis + Gunicorn
- **核心功能**: 用户认证、课程管理、视频播放（腾讯云VOD）、权限控制、播放进度追踪、安全防护
- **数据模型**: User, Course, Chapter, UserCoursePermission, PlayLog, LoginLog, LockedIP, SystemConfig, VideoFolder, VideoFile, VodEvent

#### 电商网站（mall-ecommerce）
- **技术栈**: PHP自定义框架 + MySQL + Redis + Alpine.js + Tailwind CSS
- **核心功能**: 商品浏览、SKU选择、购物车、订单管理、支付（余额/微信支付）、会员体系、收货地址、活动管理、微信OAuth
- **数据模型**: mall_users, user_addresses, categories, products, product_skus, carts, cart_items, orders, order_items, payments, wallet_ledger, activities, system_settings, system_logs, auth_lockouts, member(会员中心库), classes(会员等级)

### 1.3 设计原则
1. **统一入口**: 小程序整合课程+电商，底部TabBar切换
2. **PC端保留**: 管理员入口仅通过PC浏览器访问，小程序不提供
3. **API复用**: 后端API尽量复用现有接口，小程序作为纯前端消费API
4. **微信原生体验**: 使用微信小程序原生组件和交互模式
5. **支付集成**: 接入微信小程序支付（wx.requestPayment）
6. **内容安全**: 视频播放页实施截屏/录屏防护，保护课程版权

### 2.2 后端方案
- **复用现有API**: 课程站Flask API和商城站PHP API继续服务PC端和小程序端
- **新增小程序API层**: 在商城站PHP项目中新增小程序专用API（微信登录、小程序支付等）
- **API网关**: 通过Nginx统一路由，小程序请求走`/api/miniprogram/`前缀

### 2.3 前端技术选型
- **框架**: 微信小程序原生开发（非Taro/uni-app，保证最佳性能和原生体验）
- **状态管理**: globalData + 自定义Store模式
- **请求封装**: 统一wx.request封装，自动token管理
- **UI组件**: 微信原生组件 + 自定义组件库

**原网站对应**:

- 移动端: `mobile/play-mobile.html`

**小程序适配要点**:
- 使用`<video>`原生组件替代TCPlayer，启用同层渲染
- `enable-play-gesture`设为true，支持双击切换播放/暂停
- 播放签名通过API获取后传入
- 进度上报使用wx.request定时心跳（30秒间隔）
- 动态水印使用canvas绘制后叠加（用户信息+时间戳）

**截屏/录屏防护设计**:

微信小程序无法在所有平台完全阻止截屏/录屏，需采用分层防护策略：

| 防护层级 | Android方案 | iOS方案 | 说明 |
|---------|------------|---------|------|
| 系统级拦截 | `wx.setVisualEffectOnCapture({ visualEffect: 'hidden' })` | 不支持 | Android可完全阻止截屏/录屏，截屏时显示黑屏 |
| 截屏监听 | 系统自动拦截 | `wx.onUserCaptureScreen` 监听截屏事件 | iOS无法阻止截屏行为，仅能监听后提示 |
| 录屏监听 | 系统自动拦截 | `wx.onScreenRecordingStateChanged` 监听录屏状态 | iOS检测到录屏时暂停播放+显示遮罩 |
| 录屏预检 | 不需要 | `wx.getScreenRecordingState` 检查是否正在录屏 | iOS进入播放页时先检查录屏状态 |
| 动态水印 | 全平台覆盖 | 全平台覆盖 | 叠加半透明用户信息水印，追溯泄露源 |
| 安全提示 | 全平台覆盖 | 全平台覆盖 | 播放前弹窗提示版权保护声明 |

**具体实现逻辑**:

```
进入播放页 (onShow)
    │
    ├── Android:
    │     └── wx.setVisualEffectOnCapture({ visualEffect: 'hidden' })
    │         → 截屏/录屏时屏幕内容自动隐藏（显示黑屏）
    │
    ├── iOS:
    │     ├── wx.getScreenRecordingState() → 检查是否正在录屏
    │     │     └── 若正在录屏 → 显示安全提示遮罩，暂停播放
    │     │
    │     ├── wx.onUserCaptureScreen() → 监听截屏事件
    │     │     └── 截屏时 → wx.showToast('课程内容受版权保护，请勿传播')
    │     │
    │     └── wx.onScreenRecordingStateChanged() → 监听录屏状态变化
    │           ├── state === 'start' → 暂停播放，显示安全遮罩
    │           └── state === 'stop' → 移除遮罩，可继续播放
    │
    └── 全平台:
          └── 绘制动态水印（canvas → 临时图片 → 覆盖层）
                内容: 用户手机号(脱敏) + 用户ID + 当前时间
                样式: 半透明、斜45°、平铺覆盖

离开播放页 (onHide / onUnload)
    │
    ├── Android:
    │     └── wx.setVisualEffectOnCapture({ visualEffect: 'none' })
    │         → 恢复截屏/录屏功能，避免影响其他页面
    │
    └── iOS:
          ├── wx.offUserCaptureScreen() → 取消截屏监听
          └── wx.offScreenRecordingStateChanged() → 取消录屏监听
```

**动态水印设计**:
- 使用canvas离屏绘制水印纹理，生成临时图片作为覆盖层
- 水印内容: `用户昵称 | ID:xxx | 2026-04-09 14:30`
- 水印样式: 14rpx字号、rgba(0,0,0,0.06)透明度、-30°旋转、间距120rpx平铺
- 水印层覆盖在video组件上方，`pointer-events: none`不影响视频操作
- 每分钟更新一次水印时间戳

## 五、微信支付集成设计

### 5.1 支付流程

```
小程序端                    后端                      微信支付
  │                        │                          │
  │──1. 提交订单──────────→│                          │
  │                        │──2. 创建订单──────────→  │
  │                        │←─3. 返回order_id──────  │
  │                        │                          │
  │──4. 请求支付参数──────→│                          │
  │   (order_id, pay_method)│──5. 统一下单──────────→│
  │                        │←─6. 返回prepay_id─────  │
  │                        │──7. 签名支付参数─────→   │
  │←─8. 返回支付参数──────│                          │
  │                        │                          │
  │──9. wx.requestPayment─→│                          │
  │←─10. 支付结果──────────────────────────────────  │
  │                        │                          │
  │──11. 查询支付状态────→│                          │
  │                        │──12. 查询微信订单──────→│
  │                        │←─13. 返回支付结果─────  │
  │←─14. 返回最终状态────│                          │
  │                        │                          │
  │                        │←─15. 支付回调通知────── │
  │                        │──16. 更新订单状态        │
```

### 5.2 新增后端API

| API | 方法 | 说明 |
|-----|------|------|
| `/mall/api/miniprogram/wx-login` | POST | 小程序微信登录（code换openid+自动登录/注册） |
| `/mall/api/miniprogram/phone-login` | POST | 微信手机号快捷登录 |
| `/mall/api/miniprogram/bind-phone` | POST | 绑定手机号到微信账号 |
| `/mall/api/miniprogram/pay` | POST | 小程序支付（调用微信支付统一下单） |
| `/mall/api/miniprogram/pay-notify` | POST | 微信支付回调（小程序支付） |
| `/mall/api/miniprogram/pay-status` | GET | 查询支付状态 |
| `/mall/api/miniprogram/bind-wechat` | POST | 小程序内绑定微信openid |
| `/mall/api/miniprogram/qrcode` | GET | 获取小程序码（用于PC端扫码登录） |

### 5.3 支付参数签名

小程序支付需要以下参数传入`wx.requestPayment`:
```javascript
{
  timeStamp: '',    // 时间戳
  nonceStr: '',     // 随机字符串
  package: '',      // 统一下单返回的prepay_id
  signType: 'RSA',  // 签名类型（V3接口用RSA）
  paySign: ''       // 签名
}
```

后端需使用微信支付V3接口，配合商户API证书进行签名。

## 六、数据流设计

### 6.1 认证流程

```
小程序启动
    │
    ├── wx.login() 获取code
    │       │
    │       ├── POST /mall/api/miniprogram/wx-login {code}
    │       │       │
    │       │       ├── 后端用code换openid（微信小程序code2Session）
    │       │       │
    │       │       ├── 查找openid对应的用户
    │       │       │       │
    │       │       │       ├── 已存在 → 返回token+用户信息 → 登录成功
    │       │       │       │
    │       │       │       └── 不存在 → 返回need_bind_phone → 跳转绑定页
    │       │       │
    │       │       └── 返回结果
    │       │
    │       └── 登录成功 → 存储token到storage → 进入首页
    │
    └── token过期 → 重新走登录流程
```

### 6.2 Token管理

- 后端返回JWT token，包含user_id、openid、过期时间
- 小程序端存储到`wx.setStorageSync('auth_token', token)`
- 每次请求自动在header中携带`Authorization: Bearer <token>`
- token过期自动触发重新登录
- 课程站和商城站共享同一套用户体系（通过OpenAccountAPI打通）

### 6.3 课程站与商城站用户关联

现有架构中课程站通过`mall_api.py`调用商城站的OpenAccountAPI来管理用户。小程序端统一使用商城站的用户体系，课程站通过API验证token。

## 七、UI/UX设计规范（极简设计）

### 7.0 设计哲学

遵循微信官方设计指南核心原则——**友好礼貌、清晰明确、便捷高效、统一稳定**：

- **清晰明确**: 用户0.5秒内识别当前页面核心任务，导航路径清晰可回退
- **便捷高效**: 减少操作步骤，利用微信原生能力（授权、支付、定位等）降低输入成本
- **统一稳定**: 全局一致的视觉语言和交互模式，降低用户学习成本

### 7.1 色彩体系（极简克制）

采用极简配色方案，主色仅1个，辅助色最多2个，整体以黑白灰为基调：

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色调 | #1A1A2E | 深墨色（品牌色，用于标题、主按钮、TabBar选中态） |
| 辅助色 | #0D9488 | 青绿色（用于成功状态、课程模块标识、链接色） |
| 警示色 | #DC2626 | 红色（仅用于错误、删除等危险操作） |
| 页面背景 | #F7F7F7 | 浅灰（全局页面底色，干净留白） |
| 卡片背景 | #FFFFFF | 纯白（卡片/模块容器） |
| 主文字 | #1A1A2E | 深墨色（标题、正文） |
| 次要文字 | #999999 | 中灰（辅助信息、时间、说明） |
| 占位文字 | #CCCCCC | 浅灰（输入框placeholder、禁用态） |
| 分割线 | #EEEEEE | 极浅灰（列表分隔、模块边界） |

**色彩使用原则**:
- 一个页面最多出现2种颜色（主色+辅助色）
- 彩色仅用于需要强调的交互元素（按钮、链接、状态标签）
- 大面积区域使用黑白灰，保持视觉安静
- 禁止渐变色、禁止装饰性色块

### 7.2 字体规范（遵循微信官方）

| 用途 | 字号 | 字重 | 说明 |
|------|------|------|------|
| 页面标题 | 36rpx | 600 | 导航栏标题由系统渲染，页面内大标题用此规格 |
| 模块标题 | 32rpx | 600 | 卡片/区块标题 |
| 正文内容 | 28rpx | 400 | 列表内容、描述文字、表单标签 |
| 辅助信息 | 24rpx | 400 | 时间、备注、说明文字 |
| 标签/角标 | 20rpx | 400 | TabBar文字、极小标注 |

**字体原则**:
- 全局使用系统默认字体（微信规范要求）
- 标题与正文仅通过字重区分，不使用颜色区分
- 行高统一1.6倍，段落间距16rpx
- 单行文字超出省略，最多2行

### 7.3 布局规范（极简留白）

| 元素 | 规格 | 说明 |
|------|------|------|
| 页面左右边距 | 32rpx | 统一留白，内容不贴边 |
| 模块间距 | 24rpx | 卡片/区块之间的间隔 |
| 卡片内边距 | 32rpx | 卡片内容与卡片边缘的距离 |
| 列表项高度 | 最小88rpx | 保证触控区域≥75x75px |
| 卡片圆角 | 16rpx | 统一小圆角，不使用大圆角 |
| 按钮圆角 | 8rpx | 按钮使用更小圆角 |
| 分割线 | 1rpx solid #EEEEEE | 极细分割线，列表项之间 |

**布局原则**:
- 去除所有装饰性阴影，卡片仅用1rpx边框或背景色差区分层级
- 去除所有装饰性圆点、图标背景色块、渐变背景
- 列表左对齐，信息层级通过字号+字重表达
- 每个页面只保留一个主操作按钮

### 7.4 组件规范（原生优先）

**按钮**:
- 主按钮: 全宽、高度88rpx、主色背景#1A1A2E、白色文字28rpx
- 次要按钮: 全宽、高度88rpx、白色背景、1rpx主色边框、主色文字
- 危险按钮: 同主按钮，背景色改为#DC2626
- 禁用态: 背景色#EEEEEE、文字色#CCCCCC
- 每个页面底部最多1个主按钮+1个次要按钮

**输入框**:
- 高度88rpx、背景#F7F7F7、无显式边框
- placeholder色#CCCCCC、输入文字色#1A1A2E
- 聚焦时底部2rpx主色线条
- 表单标签在输入框上方，间距8rpx

**列表**:
- 左侧主信息、右侧辅助信息（金额、箭头、状态）
- 列表项之间1rpx分割线
- 无装饰性图标背景色块

**卡片**:
- 白色背景、16rpx圆角、无阴影
- 内容区32rpx内边距
- 卡片间距24rpx

**TabBar**:
- 遵循微信原生TabBar样式，不自定义
- 图标: 线性风格、未选中#999999、选中#1A1A2E
- 文字: 20rpx、颜色同图标
- 4个Tab: 首页、课程、商城、我的

**弹窗/半屏面板**:
- SKU选择、地址选择等使用半屏弹出面板（`<page-container>`或自定义组件）
- 面板圆角顶部24rpx、高度屏幕60%-70%
- 遮罩层rgba(0,0,0,0.5)

### 7.5 交互规范（微信原生体验）

- **下拉刷新**: 仅首页和列表页启用，使用微信原生下拉刷新样式
- **上拉加载**: 长列表使用`<scroll-view>`的`bindscrolltolower`触发加载
- **操作反馈**: 成功用`wx.showToast({ title: '操作成功', icon: 'success' })`，失败用`wx.showToast({ title: '错误信息', icon: 'none' })`
- **二次确认**: 删除、取消订单等危险操作使用`wx.showModal`
- **页面导航**: `wx.navigateTo`（可返回）、`wx.switchTab`（TabBar页）
- **图片预览**: `wx.previewImage`
- **复制**: `wx.setClipboardData`，复制后自动toast提示
- **空状态**: 居中图标+文字+操作按钮，不使用大段描述
- **加载态**: 骨架屏（首页、列表页），局部loading（按钮loading态）
- **表单验证**: 实时校验，错误信息显示在输入框下方，红色文字24rpx
- **页面转场**: 使用微信默认转场动画，不自定义

## 十一、安全设计

### 11.1 通信安全
- 所有API请求强制HTTPS
- 请求签名验证（timestamp + nonce + signature）
- Token有效期2小时，自动续期

### 11.2 支付安全
- 订单金额后端校验（不信任前端传入的金额）
- 支付回调签名验证
- 幂等性处理（同一订单不重复支付）

### 11.3 视频安全
- 播放签名有效期1小时
- 设备并发控制（最多2台设备同时播放）
- 播放进度反刷校验
- 每小时访问次数限制
- **截屏/录屏防护**（多层防御策略）:
  - **Android硬防**: `wx.setVisualEffectOnCapture({ visualEffect: 'hidden' })`，截屏/录屏时屏幕内容隐藏，显示黑屏。页面离开时必须恢复为`none`
  - **iOS软防**: 
    - `wx.onUserCaptureScreen` 监听截屏，截屏时toast提示版权声明
    - `wx.onScreenRecordingStateChanged` 监听录屏状态，录屏开始时暂停播放+显示安全遮罩
    - `wx.getScreenRecordingState` 进入播放页时预检录屏状态，若正在录屏则阻止播放
  - **全平台动态水印**: canvas绘制用户信息（脱敏手机号+用户ID+时间戳），半透明斜45°平铺覆盖视频区域，每分钟刷新时间戳，追溯泄露源头
  - **版权声明**: 播放前弹窗提示"本课程内容受版权保护，禁止截屏、录屏或传播"，用户确认后方可播放
  - **安全日志**: 截屏/录屏事件上报后端记录（用户ID、时间、章节ID），用于安全审计

### 11.4 用户数据安全
- 手机号脱敏显示
- openid不暴露给前端
- 密码修改需验证旧密码

## 十三、注意事项

1. **小程序包大小**: 主包控制在2MB以内，课程播放页和商城详情页可考虑分包加载
2. **视频播放**: 微信小程序`<video>`组件支持同层渲染（基础库2.4.0+），需设置`enable-play-gesture`优化交互
3. **富文本**: 使用rich-text组件，需对HTML进行安全过滤和样式适配，过滤script/iframe等危险标签
4. **分享**: 商品详情页和活动详情页支持分享到聊天，定义`onShareAppMessage`返回标题+路径+封面图
5. **订阅消息**: 订单状态变更（支付成功、发货、完成）使用订阅消息通知，需先调用`wx.requestSubscribeMessage`
6. **小程序码**: PC端可展示小程序码引导用户扫码进入，使用`wxacode.getUnlimited`接口生成
7. **版本更新**: 使用`UpdateManager`检测小程序更新，提示用户重启更新
8. **网络状态**: 弱网环境下的友好提示和重试机制，使用`wx.onNetworkStatusChange`监听
9. **微信登录合规**: 不得强制要求用户授权手机号才能使用基础功能，先静默登录再按需获取手机号
10. **隐私协议**: 首次启动需弹出隐私协议弹窗，用户同意后方可调用微信API，使用`wx.getPrivacySetting`检测
11. **用户信息获取**: `wx.getUserProfile`已废弃，头像昵称通过`<button open-type="chooseAvatar">`和`<input type="nickname">`获取
12. **截屏录屏防护生命周期**: `wx.setVisualEffectOnCapture`必须在页面`onHide`/`onUnload`时恢复为`none`，否则影响其他页面
13. **iOS录屏监听**: `wx.onScreenRecordingStateChanged`监听的是设备全局录屏状态，需在页面离开时`wx.offScreenRecordingStateChanged`取消监听
14. **iOS录屏预检**: 进入播放页时必须调用`wx.getScreenRecordingState`检查是否已在录屏，此API仅iOS有效
15. **动态水印性能**: 水印canvas绘制不宜过于频繁，建议1分钟刷新一次，避免影响视频播放性能
16. **小程序审核**: 视频类小程序需提供网络视听许可证或相关资质，支付类需提供相关资质
17. **深色模式**: 建议在`app.json`中配置`darkmode: true`，提供深色模式适配
